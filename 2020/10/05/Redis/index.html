一、Redis简介

1、redis解决的问题

网站出现崩溃的原因：
	1、海量用户
	2、高并发
罪魁祸首————关系型数据库
	性能瓶颈：磁盘IO性能低下
	扩展瓶颈：数据关系复杂，扩展性差，不便于大规模集群
解决思路：
	降低磁盘IO次数，越低越好 ———— 内存存储
	去除数据关系，越简单越好 ———— 不存储关系，仅存数据

2、Nosql

NoSQL：即Not-OnlySQL(泛指非关系型的数据库)，作为关系型数据的补充
作用：应对基于海量用户和海量数据前提下的处理问题
特征：
	1、可扩容、可伸缩
	2、大数据量下高性能
	3、灵活的数据模型
	4、高可用
常见的NoSQL数据库：
  Redis
  memcache
  Hbase
  MongoDB

3、解决方案（电商场景）

——MySQL：
		1、商品基本信息
		2、价格
		3、厂商
——MongoDB：
		1、商品描述
		2、商品详情
		3、评论
——分布式文件系统：
		1、图片信息
——Lucene、Solr：
		1、搜索关键字
——Redis：
		1、热点数据

4、Redis简介

概念：Redis(REmote DIctionary Server)是用C语音开发的一个开源的高性能键值对（key-value）数据库
特征：
	1、数据间没有必然的关联关系
	2、内部采用单线程进行工作
