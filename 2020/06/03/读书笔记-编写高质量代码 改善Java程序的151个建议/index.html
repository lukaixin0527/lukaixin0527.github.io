第一章、Java开发中通用的方法和准则

建议1：常量和变量中不要出易混淆的字母

包名全部小写，类名首字母全大写，常量全部大写并用用下划线分割，变量采用驼峰命名法命名。
不要出现 long = 1l; 数字"1"和字母"l"容易混淆。

建议2：莫让常量蜕变成变量

定义常量时，在编译器就必须确定其值，不应该在运行期更改
例如：public static final dateTime = new Date();

建议3：三元操作符的类型务必一致

    public static void main(String[] args) {
        int i = 80;
        String s = String.valueOf(i < 100 ? 90 : 100);
        String s1 = String.valueOf(i < 100 ? 90 : 100.0);
        System.out.println(s.equals(s1)); // false
    }

若两个操作数不可转换，则不做转换，返回值为Object类型
若两个操作数是明确类型的表达式（比如变量），则按照正常的二进制数字来转换，int类型转换成long类型，long类型转换成float类型等。
若两个操作数中有一个是数字S，另一个是表达式，且其类型标识为T,那么若数字S在T范围内，则转换成T类型，若S超出T类型的范围，则T转换成S类型
若两个操作数都是直接量数字，则返回值类型为范围较大者。

建议4：避免带有变长参数的方法重载

建议5：别让null值和空值威胁到变长方法

建议6：覆写变长方法也循规蹈矩

建议7：警惕自增的陷阱

    public static void main(String[] args) {
        int count = 0;
        for (int i = 0; i < 10; i++) {
            count = count ++;
        }
        System.out.println(count); // 0
    }

解析：i++表示的是先赋值后加1，++i是先加1后赋
count++是一个表达式，是有返回值的，他的返回值就是count自加前的值，java对自加是这样处理的：首先把count的值（注意是值，不是引用）拷贝到一个临时变量区，然后对count变量加1，最后返回临时变量区的值。
  
      public static void main(String[] args) {
        int count = 0;
        for (int i = 0; i < 10; i++) {
             count++;
        }
        System.out.println(+count); // 10
    }

建议8：编程中尽量不要出现旧语法，如果时间过久会困扰(看不懂)编程。

建议9：少用静态导入
