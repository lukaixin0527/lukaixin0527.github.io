Nginx基础知识

Nginx基础

一、什么是Nginx？

1、nginx反向代理、代理服务器、静态服务器，

2、nginx作用：集群(减轻单台服务器压力)、反向代理(不暴露真实IP地址)、虚拟服务器、静态服务器(动静分离)

3、nginx优点：性能优秀，同时5wan并发，内存小，轻量级服务器

4、安全架构问题，如何解决

  • nginx反向代理(不暴露真是IP地址)
  • 使用Https防止抓包分析Http请求
  • 搭建企业黑名单和白名单系统（防盗链）
  • 模拟请求（csrf 业务攻击）、XSS、sql注入
  • ddos（流量攻击，占用宽带，影响用户正常使用）
  • CDN加速

5、反向代理服务器

​ lvs、F5、haproxy

三、Nginx实现反向代理

1、什么是反向代理？

客户端——> 反向代理服务器nginx ——> tomcat服务器集群

nginx和tomcat属于同一个网络，nginx向外暴露外网ip,采用负载均衡机制（权重、轮询、IP绑定），调用tomcat服务集群，nginx和tomcat之间的通信属于内网访问，保证了tomcat的安全性。

2、 在集群会产生什么情况？

  • 分布式job幂等性(重复)问题，使用XXLjob分布式任务调度平台
  • 回话共享问题
  • 分布式生成全局ID （比如生成订单号，使用时间戳，集群情况下会出现幂等性问题。解决方案：提前生成好订单号，放到redis缓存中，或者使用分布式锁，同时只能一个执行）

3、配置反向代理

location / 表示拦截所有请求

{  listen       80;
    server_name  localhost;    

    location / {          
        proxy_pass   https://www.baidu.com;
        index  index.html index.htm;
}

监听 localhost:80端口，location/ 拦截所有请求，然后跳转到www.baidu.com，默认访问index.html页面

四、负载均衡

负载均衡：提高网站吞吐量（接收响应），减少单台服务压力

nginx负载均衡 算法、轮询机制（服务器性能一样）、权重（服务器性能不一样）、IP绑定（只让一个IP地址进行访问，解决会话共享问题）

1、轮询机制

upstream backserver { 
server 192.168.0.14; 
server 192.168.0.15; 
}
{  listen       80;
    server_name  localhost;    

    location / {
        proxy_pass  https://backserver;
        index  index.html index.htm;
}

2、权重机制

upstream backserver { 
server 192.168.0.14 weight=5;  // 只是比例关系，不是次数问题，其实是1:2
server 192.168.0.15 weight=10; 
}

3、IP绑定

upstream backserver { 
ip_hash; 
server 192.168.0.14; //一个IP地址会默认访问14
server 192.168.0.15; 
}

五、服务器宕机容错机制 高可用

1、服务器宕机怎么解决？

nginx服务器：一主一备或者多主多备用

如果主备服务器都宕机了，使用高可用工具keepalive重启脚本，自动重启。

重启N多次后还是失败后，发送报警邮件。

如果tomcat服务器集群，有一个宕机了怎么解决？

server {
       listen       80;
       server_name  www.itmayiedu.com;
       location / {
	          proxy_pass  http://backserver;
		        index  index.html index.htm;
         
		      proxy_connect_timeout 1; // 超时1秒后轮询到下一台服务器
           proxy_send_timeout 1;
           proxy_read_timeout 1;
       }
	}

2、如何优雅的发布版本？

逐步更新版本，一主一备的更新。

3、tomcat发布版本如何解决session失效问题？

存放到redis中，或者持久化到数据库中

六、使用Nginx搭建企业级API接口网关

1、跨域问题

属于前端ajax访问请求域名与浏览器访问的域名地址不相同，存在跨越问题

如何解决？

  • jsonp 不支持post请求，支持get
  • httpclient进行内部转发
  • 使用http响应头允许跨域设置
  • 使用nginx搭建企业api接口网关
  • 使用SpringZULL接口网关

2、nginx搭建企业api接口网关原理

企业API接口网关原理：通过域名相同，项目不同，进行拦截，跳转到真是服务器中。

nginx接口网关，拦截所有请求，根据请求不同进行分发。（权限控制）

例:https://www.project.com/A 访问，nginx拦截，转发到 tomcat-A服务器 http://localhost:8080/A

3、nginx搭建网关

server {
        listen       80;
        server_name  www.project.com;

        location /A {
		    proxy_pass   http://localhost:8080/A;
			  index  index.html index.htm;
        }

		    location /B {
		    proxy_pass   http://localhost:8080/B;
			  index  index.html index.htm;
        }
    }
